# Lucky Block — Developer Onboarding

Welcome to the Lucky Block project. Read this before touching any code.

## Step 1 — Understand the Architecture
Read architecture.md first.

## Step 2 — Understand Your Role
Read the relevant section in the main README.

## Step 3 — Read the API Contract
Read api-contract.md. This is how the game and backend communicate.

## Step 4 — Clone Your Repo
Clone only the repo relevant to your role.

## Step 5 — Branching Rules
- Never push directly to main
- Never push directly to develop
- Always create a feature branch
- Branch naming: feature/your-feature-name

## Step 6 — Communication Rules
- API changes must be updated in api-contract.md first
- Open a GitHub Issue for every bug found
- Weekly sync with full team
